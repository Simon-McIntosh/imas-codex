§1 — Problem framing and settled design rules

Plan 41 attacks the physical-base explosion directly: too many compound bases in ISN prevent qualifier extraction and fracture catalog grouping. The survey marks the plan at 0.50 because the governing grammar rules are already clear, but the rollout through ISN, codex, iterative bootstrap, and export is still open.

  • Insertion-order qualifiers are mandatory. Qualifiers must preserve parse order rather than being alphabetized during compose.
  • Subjects win over qualifiers. Compound subject tokens stay atomic and qualifier stripping happens only after subject removal.
  • Processes stay suffix-only. Process vocabulary is only a _due_to_… template; the same token used as a prefix is treated as a qualifier.
  • Multi-word qualifiers remain atomic. Tokens such as center_of_mass must not be broken apart by the generic _of_ operator logic.
  • The base-count target is moderate, not extreme. The plan explicitly moved from an over-aggressive 45-base target to a validated 50–80 dimensional-base range.

§2 — Open rollout work

  • Phase A — ISN grammar redesign. Create qualifiers.yml, reduce physical_bases.yml to dimensional quantities, add missing process tokens, load qualifiers from YAML in the parser, emit ordered qualifiers in compose, prove 100% round-trip success on the 479-name corpus, and cut an ISN RC release.
  • Phase B — codex pipeline update. Bump the ISN dependency in both pyproject locations, refresh grammar context and prompts, clear the SN graph, and run the first bounded rotation with the new grammar.
  • Phase C — iterative bootstrap. Run repeated low-cost rotations, inspect vocab gaps and parse errors, add qualifier/process tokens upstream, cut new RCs, and re-run until the grammar stops shedding obvious gaps.
  • Phase D — catalog/export closure. Export validated names, preview grouping behaviour, and publish only after the new dimensional grouping reads cleanly in the renderer.

§3 — Risks and validation gates

  • The offline gate is non-negotiable. Phase A requires a 100% round-trip result on the 479-name corpus before graph work proceeds.
  • Subject/qualifier ambiguity remains the main semantic risk. The plan mitigates this by keeping subject parsing earlier and refusing to let the same token behave as both subject and qualifier inside one name.
  • Wrong qualifier ordering is a regression risk. The plan treats ordering as a first-class round-trip property, not a formatting detail.
  • Bootstrap cost is intentionally bounded. The iterative rotation design exists so vocab mistakes can be corrected cheaply between runs instead of after a full expensive regeneration.
  • Documentation updates remain open. AGENTS guidance, ISN guidance, shared grammar references, and the auto-generated schema reference all need updates once the rollout actually lands.

§4 — Migration note

The important thing to preserve from the markdown source is that plan 41 is not an architectural rewrite. It is a vocabulary reorganization whose required upstream redesign was already present by v0.8.0rc31; this write-back closes the codex-side adoption and hard-blocker cleanup while explicitly carrying preview/render grouping follow-on validation downstream.
