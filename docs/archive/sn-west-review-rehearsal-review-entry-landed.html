§3 review-entry — refine products stranded at name_stage=pending never entered review (2026-07-23)

Gap-closure left ~37 WEST sources carrying a correct, valid produced name that gap-only runs reported as no_eligible_work: the names existed and validated but sat at name_stage=pending (score null) and were never claimed by REVIEW_NAME, which claims drafted. This was a real pipeline transition defect, not a naming gap. Root-caused, fixed with TDD, and the stranded set was advanced into review and accepted on merit.

Root cause — refine adopts a placeholder without advancing its stage

A refined name is minted as a NEW node keyed by the name string. The normal compose path advances a name to drafted with an unconditional MATCH … SET (_finalize_generated_name_stage), so compose never strands. But persist_refined_name initialised the successor stage with MERGE (new) ON CREATE SET name_stage='drafted'. When a refine converges onto a name string that already exists as an un-reviewed placeholder — a locus/parent scaffold minted pending by the HAS_PARENT derivation writer, or a bare MERGE-created node with no stage — ON CREATE does not fire, so the successor kept the placeholder stage even though the refine migrates all its PRODUCED_NAME / HAS_STANDARD_NAME sources onto it and supersedes the predecessor. The only code path that writes name_stage='pending' is that ON-CREATE derived-parent MERGE.

Evidence (live graph): every one of the 22 WEST pending pipeline names carrying a produced source was a refine product (had a REFINED_FROM out-edge). A minimal reproduction — refine an old name into a pre-existing pending placeholder — left the successor pending before the fix and drafted after (test_refine_into_pending_placeholder_advances_to_drafted).

Fix (TDD, commit 35e46bc7)

ChangeMechanism
persist_refined_name — advance a placeholder-merged successor at write time Added ON MATCH SET: when the merged-onto node is an un-reviewed placeholder (name_stage null or 'pending') and not a structural derived parent, advance it to drafted + origin='pipeline'. Live/terminal stages (drafted/reviewed/accepted/…) and derived parents are left untouched.
reconcile_reviewable_name_stage — self-heal already-stranded names New reconcile in graph_ops: any name at name_stage null/'pending' that is valid, not derived, and carries a non-derived produced source is advanced to drafted. Idempotent; wired into the run_sn_pools startup reconcile so it runs every sn run (incl. --only reconcile), before the pools, so the review pool sees the advanced names the same run.

Both are guarded to never touch a structural derived parent (those skip REVIEW_NAME and are accepted structurally by ENRICH_PARENTS) — the review-name claim already excludes origin='derived'. No name_stage is ever hand-set in Cypher: advancing pending → drafted feeds the RD-quorum, which scores on merit.

Tests

New tests/standard_names/test_reviewable_name_stage.py — a fast import/return-shape guard plus live-graph (@pytest.mark.graph) end-to-end cases: refine into a pending placeholder advances to drafted; refine into a derived parent placeholder stays pending; refine into a brand-new name is drafted (unchanged); refine into a live accepted name is not demoted; and the reconcile advances a stranded name, skips derived/quarantined names, and is idempotent. Four pre-existing tests that enumerate the startup reconciles in a patch.multiple mock list were extended for the new reconcile. Full SN suite green.

Outcome — stranded names accepted on merit

The reconcile_reviewable_name_stage pass advanced every stranded name (22 WEST refine products, plus a graph-wide backlog) into drafted, and a cost-capped names-only review pass (sn run --only review --names-only --cost-limit 12, $12.61, 86 reviewed / 22 refined, stop_reason=budget_exhausted) accepted them on merit through the RD-quorum. WEST accepted sources rose 290 → 306 / 355 (+16). Crucially, the stranding signature is gone: 0 valid pipeline names carrying a produced source now sit at pending graph-wide (was 22).

The WEST residual after the pass is 29 names that all entered review (the fix working) — 18 reviewed below the 0.85 accept bar (routed to refine, e.g. magnetic_shear 0.825, plasma_confinement_time 0.475) and 11 drafted still queued when the budget cap cut the pass short — plus 2 genuinely grammar-rejected/quarantined names (cross_section_of_flux_surface, flux_surface_average_magnetic_field_magnitude) that are correctly excluded from review (they regenerate, not review). None are stranded; the remainder is ordinary review throughput a subsequent pass drains. The genuinely-hard tail (gm-metric coefficients, hard_xray_width, the camera up-vector) scores low on merit and is a naming-quality matter, not a transition defect.

What's next

The review-entry throughput matter is closed — refine products now enter review at write time and any historical straggler self-heals on the next run. Stage 2 (cut the fork-PR batch RC, f-swrr-007) is unblocked: the WEST accepted set now includes the previously-stranded names.
