DD Node Kind Classification — Audit & Remaining Work

This archive keeps the audit trail for the first large DD category migration. It is important because the markdown source recorded both what had already landed and the exact graph mutations that were applied before later plans absorbed the work.

§1 — Landed outcomes captured by the audit

  • Schema and classifier work landed. The archive preserves the creation of node_classifier.py, node_categories.py, and the initial enum expansion beyond the old data/error/metadata split.
  • Consumers were updated. The markdown recorded the wide import migration across build, search, CLI, and graph code.
  • The live graph was actually migrated. The source documented the shadow-property classification, full coverage validation, atomic swap, and stale-embedding cleanup that left only true quantity nodes embedded.
  • Resulting state was measured. The post-migration counts and the “zero non-quantity embeddings” invariant are preserved here as the historical checkpoint.

§2 — Critical defects that remained visible

The audit mattered because it did not treat the migration as finished. It kept two high-confidence bugs front and center:

  1. Wrong relationship direction in pass 2. PARENT_OF was queried even though the graph used HAS_PARENT, demoting about 1,703 structure-with-unit nodes that should have stayed data-bearing.
  2. Over-broad coordinate reclassification. Any HAS_COORDINATE target became coordinate, which incorrectly swallowed hundreds of grid-stored physics values such as .../values, .../radial, and .../measured paths.

The source also preserved the important non-bug: the apparent identifier-count mismatch was explained by ids_properties/occurrence_type nodes that correctly remained metadata.

§3 — Why this file still matters

The later DD Unified Classification & Enrichment Pipeline plan absorbed this work at a broader architectural level, but this archived note remains the best record of the original graph repair, the measured category distribution, and the lessons learned about relationship direction, path-aware coordinate handling, and writing tests before applying graph one-offs.
