Standard-Name Catalog Approval Pipeline — landed record (2026-07-21)

Six phases building the first human-review gate for the machine-generated standard-name set: drive eligible WEST DD sources to accepted names, PR an additive batch upstream for expert review, and freeze approved names back into the local graph. All decisions were locked before implementation; every phase landed test-first with graph-marked tests run against the live Neo4j.

P1 — gap-only focus seeding

sn run --focus <file> defaulted to reset-all: it force-reset every focused path's name to pending, churning names the catalog already held. Now the default is gap-only — a focused path carrying a live accepted/approved name is skipped; --reseed restores reset-all. New graph helper partition_focus_by_accepted(paths) → (gap, accepted) drives the filter in the CLI focus block.

P2 — two schemas & minting the SN set

Extraction speaks DD paths; export/PR/merge speak SN ids. A kind discriminator (sn_sources | sn_names) now tags both schemas; a new sn_names.schema.json defines the frozen batch record. load_focus_file dispatches by kind (raising if absent). mint_sn_list(dd_paths) resolves paths to the review set: a source_paths base join (bare DD paths, not dd:-prefixed) then an immediate-family closure — HAS_PARENT parent, that parent's direct children (siblings), and own direct children; one hop, non-transitive; dead names excluded; grammar-base / cluster / lineage are deliberately not closure edges. Unmatched paths are reported, not dropped.

P3 — additive review-batch export

_fetch_candidates(batch=…) switches the name gate to name_stage = 'approved' OR id ∈ batch (still validation- and docs-gated), so a review PR carries the already-approved catalog plus exactly the batch — an additive diff, no phantom removals. run_export stamps export_scope: review and the sorted review_batch id-set into catalog.yml; normal RC exports are byte-identical to before. The ISN StandardNameCatalogManifest model was extended (P5) so the marker validates cleanly; imas-codex's best-effort manifest validation writes it regardless.

P4 — one sn release --review-batch command

run_review_release folds mint → freeze artifact → export → branch → push(fork) → gh pr create → back-fill PR number/URL into a single call so the artifact, the pushed RC catalog, and the PR stay in lock-step. The frozen sn-names batch record lives under manifests/reviews/<rc>.sn_names.yaml, tagged by the self-assigned RC version (the traceable key, known before push); the PR number is back-filled after creation. Export / publish / PR steps are injectable, so the flow is tested against a local bare repo with no live GitHub call.

P5 — SPA fixed review view (ISN repo)

In imas-standard-names: the manifest model carries review_batch and export_scope: review; build_site_dataset emits a top-level review_batch into data.json when the manifest sets it; the SPA, seeing that marker, renders a fixed view of exactly those names (no show-all toggle), reusing the existing Filters/ResultsList machinery. Normal builds are unchanged. The RC version URL is the shareable per-batch link. Landed on the ISN fork (ddf76ef, 411d812) with 4 pytest + 4 vitest tests.

P6 — sn merge state machine & the contested stage

Per batch name in a merged PR: untouched names auto-promote accepted → approved (the human approved by merging — an implicit compliance rubber-stamp; needs complete PR metadata and the --batch artifact); edited names re-trigger the full review — pass → approved, fail → the new contested stage.

contested is a NameStage holding state: neither published nor silently reverted, frozen and excluded from every pool (added to all NOT IN ['superseded','exhausted'] claim + child/parent-liveness predicates; the positive dead-parent-rewire select deliberately excludes it). Resolution is human-driven and on the record: sn edit → re-review; sn approve <id> --override --reason (force approve over the rubric); sn revert <id> --reason (drop the edit back to accepted). sn status --contested lists them with the failing verdict. Schema gained contested plus contested_reason/contested_at/contested_resolution and the previously-undeclared merge_quarantine_reason/_at.
