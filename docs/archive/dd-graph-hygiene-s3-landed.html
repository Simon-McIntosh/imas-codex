Root cause

Empty-string descriptions were not a data accident — they were written by the pipeline. Two source defects, same family:

  • Clobber via coalesce('', existing). The StandardName / StandardNameSource merge sites used coalesce(incoming, existing) to preserve prior text. But an empty string is non-null, so coalesce('', existing) returns '' — a blank incoming description silently overwrote a real one.
  • Seeders emitted ''. Source seeders (including the --focus re-seeder in the CLI) built the source dict with a literal "description": "" when a DD path had no documentation, so a source could be born empty-string rather than null.

What landed

ChangeWhere
Wrap the incoming description in nullIf(…, '') at both merge chokepoints — a blank never overwrites existing text and a source is never born with ''standard_names/graph_ops.py, loop.py, pool_adapter.py (7a6eb498)
Pass None (not "") from the --focus re-seedercli/sn.py (369b91c4)
