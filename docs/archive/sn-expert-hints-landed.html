§6 demo — the full story (including the failure that improved the tool)

Graph reality at demo time: area_of_plasma_boundary (DD equilibrium/time_slice/global_quantities/area, the poloidal cross-sectional area of the LCFS) was superseded with no successor — its own review had killed it on precisely the ambiguity this plan targets ("ambiguous between cross-sectional area enclosed by the boundary and toroidal boundary surface area", review disagreement, mean 0.846). The DD path had no live standard name.

Attempt 1 — rejected, and the rejection was diagnostic gold

sn edit area_of_plasma_boundary --rename poloidal_cross_section_of_plasma_boundary --reason … --scope self worked mechanically (redesign skipped generate, successor drafted, reason rode into the 3-cycle RD-quorum). The quorum scored it 0.675 — with the grammar reviewer hallucinating that the registered base should be cross_sectional_area (unregistered; the candidate parses cleanly as projection=poloidal + base=cross_section). The refine worker then followed the reviewer's wrong suggestion into a deterministic validation wall twice and exhausted the name, and that exhaust path bypassed the edit_status lifecycle hook. This was the §3.2 revert pull observed live, in a form the plan had not predicted: the reviewer inventing vocabulary.

Three fixes landed as 85f45373:

  • Rename successors are stamped with their parsed ISN segment fields (physical_base, geometry, grammar version) at edit time.
  • The review claim returns those fields, and the review prompt's steering block adds a deterministic grammar check line: the candidate PARSES, with its verified decomposition — score grammar from evidence, not assumption.
  • The refine deterministic-exhaust paths now close an open edit (edit_status open→exhausted), axis-matched.

Attempt 2 — accepted, docs and all

sn edit poloidal_cross_section_of_plasma_boundary --rename cross_section_of_plasma_boundary --reason … (the registered-base form; reviewer pressure legitimately steered the final shape). Scoped rotation sn run --flush --scope-run-id sn-edit-20260703T104602Z:

  • Name accepted 0.988 (grammar 20/20, semantic 19, convention 20, completeness 20) — the reviewer explicitly cited the deterministic grammar check and the swept-surface disambiguation. edit_status → applied.
  • Docs generated and accepted 0.925 in the same rotation — the documentation states the poloidal-plane region bounded by the LCFS, resolving the original ambiguity in the catalog text itself.
  • Chain: cross_section_of_plasma_boundary → poloidal_cross_section_of_plasma_boundary [superseded] → area_of_plasma_boundary [superseded]; source provenance (PRODUCED_NAME) migrated; rotation cost ≈ $0.72.

Control — the gate stayed honest

Deliberately-bad edit: sn edit major_radius_of_secondary_x_point --rename toroidal_angle_of_secondary_x_point with a specious reason, given the SAME anti-revert directive and the SAME grounded grammar check. Verdict: 0.825 — below the 0.85 gate, grammar 20/20 (it does parse) but semantic 12/20: "the supplied unit is m, which is physically inconsistent with an angle". Rejection came from physics, exactly where it should. The control was then retired (exhausted/rejected) so no rotation refines it.

Deviations & residuals

  • No-rollback gap: a rename edit supersedes the target before review; a rejected/exhausted edit leaves the predecessor superseded without an accepted successor (harmless for the orphan targets used here; risky for accepted targets) — queued as a followup.
  • The control's DD-source PRODUCED_NAME provenance now points at the retired control name (was pointing at a dead orphan before) — cosmetic, catalog export only emits accepted names.
  • Hint mode and docs mode are unit-tested (183 tests) but were not exercised in the live demo; the subtree cascade likewise (the demo target is a leaf).
  • Grammar grounding currently applies to steered candidates only; generalizing it to all pool reviews is a candidate follow-on (the hallucination it fixes is not edit-specific).
