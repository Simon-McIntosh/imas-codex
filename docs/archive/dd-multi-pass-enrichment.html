DD Multi-Pass Enrichment

This archive preserves the second-pass DD enrichment design that was meant to sit on top of DD Unified Classification & Enrichment Pipeline. The source plan argued that sonnet-quality pass-1 descriptions were already valuable, but that sibling and peer descriptions would make the next pass materially better.

§1 — Core design preserved

  • New lifecycle step: built → enriched → refined → embedded.
  • New worker: refine_worker between enrich and embed, with the embed worker updated to claim refined nodes instead of enriched.
  • Natural sibling barrier: a node only becomes claimable for refinement once its enrichable siblings are all past built, avoiding a coarse global barrier.
  • New context sources: sibling descriptions, same-cluster peer descriptions, cross-IDS duplicates, and the node’s own pass-1 description.
  • Idempotency boundary: a dedicated refinement_hash keyed from pass-1 text, peer context, and model choice.

§2 — Why the plan mattered

The durable idea is that DD descriptions become more consistent when they can see one another. The markdown source preserved three concrete failure modes in single-pass generation: ambiguous sibling families, cross-IDS wording drift, and inconsistent terminology among semantic-cluster peers. This archive keeps that reasoning and the explicit cost model: roughly two passes of sonnet enrichment, about $100 total, and about six hours for a full rebuild.

§3 — Dependencies and follow-on work

The markdown source was explicit that this work depended on the classifier and status-machine changes from DD Unified Classification. It also preserved the later extensions that were intentionally left for follow-up: selective auto-promotion when no useful peer context exists, targeted re-refinement for bad descriptions, and a possible third summarization pass for structure nodes.
