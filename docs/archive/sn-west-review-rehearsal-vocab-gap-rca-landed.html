§3 stage 1 — vocab-gap signal loss: two defects found and fixed (2026-07-22)

Draining the WEST mop-up surfaced a class of blocked sources parked at vocab_gap with no record of what blocked them. Pulling that thread revealed two independent defects — one in how the outcome is persisted, one in how the gap is classified — that together let the pipeline fabricate vocabulary gaps for grammar that already exists and then silently strand the sources. Both are fixed; the false gaps are reconciled away.

Defect 1 — the outcome was persisted lossily

When compose reports a vocab gap it hands the marker a list of {source_id, segment, token, reason}. The live marker (_update_sources_after_vocab_gap) set only status='vocab_gap' and discarded the token — so a blocked source recorded nothing about why. The gap survived only as a global VocabGap node hung off the DD IMASNode, recoverable by a join and, for sources whose gap was non-actionable, absent entirely: 9 of 38 WEST sources had no gap record anywhere.

Worse, the marker filtered only open segments while the node-writer (write_vocab_gaps) drops five non-actionable categories. So a source whose reported gap was decomposable, wrong_slot_placement, ambiguous_known_token, or a false_positive — i.e. still nameable, the composer merely erred — was retired to vocab_gap and stranded, with the matching VocabGap node never written.

The fix records skip_reason='vocab_gap' + skip_reason_detail (the blocking segment:token list) so a source explains itself, and retires only on a genuinely-absent closed-segment gap; non-actionable reports keep the source retryable under the attempt-count cap. A shared is_actionable_gap() means the node-write and source-mark paths classify identically and cannot drift; the dead mark_sources_vocab_gap is removed.

Defect 2 — the classifier was blind to operators (the real root cause)

Reviewing the actual blocking tokens against the ISN grammar showed the biggest "gaps" were not gaps at all. ISN operators — flux_surface_averaged, line_integrated, normalized, square, derivative_with_respect_to, gradient — are a grammar mechanism separate from SEGMENT_TOKEN_MAP: they compose into names as <op>_of_<base> or as a postfix. Codex's gap classifier (is_known_token/classify_gap) never consulted the operator vocabulary, so a composer that mis-slotted a known operator as a qualifier had it classified absent — fabricating a gap and, before Defect 1's fix, retiring the source.

is_known_token now augments its result with a synthetic operator membership derived at runtime from get_grammar_context().grammar.vocabularies.operators. A mis-slotted operator classifies as wrong_slot_placement — no VocabGap node, source stays retryable. Reconcile then reclassified the standing false gaps in place: normalized (n=43), square (n=14), flux_surface_averaged (n=5), line_integrated — all absent → wrong_slot_placement; 15 gaps reclassified, sources revived to extracted.
