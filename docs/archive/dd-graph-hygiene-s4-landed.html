Root cause

The build's stale-label clear pass (build_dd.py, ~line 1974) set cocos_transformation_type = null on paths absent from the current DD horizon but left cocos_label_source set — a half-state (source with no type) invisible to COCOS-filtered queries. All 20 affected nodes were lifecycle_status='removed' historical fields (e.g. summary/local/separatrix_average/velocity_tor/<species>), exactly the paths the clear pass drops.

What landed

ChangeWhere

Applied to the live graph: 20 half-state nodes groomed. The inverse invariant (TestCOCOSBackfill.test_cocos_label_source_populated: type ⇒ source) still holds — the groom only touches type-null nodes. Sibling COCOS test classes green.

Consistency note. This complements the locked dd-horizon-update decision D-historical = current-only (removed nodes are provenance-only): rather than tolerate a dead half-state, the groom removes it so no node — current or historical — carries a meaningless source tag.
