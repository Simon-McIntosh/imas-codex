Plan 38 — Grammar vNext (archived)

This archive preserves the last markdown version of the proposed Standard Name grammar overhaul. The source document was not a landed implementation record: it was a design package assembled after two reviews rejected plan 37’s preposition-stripping direction and converged on a larger grammar reset.

§1 — What this plan settled

  • Plan 37 was superseded. The archive keeps the explicit decision that identity markers and prepositions should remain visible rather than being stripped for clustering convenience.
  • The rc20 corpus preservation step had already landed. The plan records v0.2.0-rc20-corpus as the frozen 479-name baseline used for later vocabulary mining and round-trip checks.
  • The core grammar proposal was the 5-group IR. Operators, projection, qualifiers, base, locus, and mechanism were re-framed as typed derived structure rather than a loose collection of partially overlapping segments.
  • Closed vocabularies became the governing rule. The archived design keeps the move toward typed coordinate axes, a typed locus registry, a closed physical_base vocabulary, and a unified operator registry.

§2 — Key codex-side insight worth preserving

The most durable codex decision in the markdown plan was not a prompt tweak but a storage rule: if the grammar parser becomes round-trip safe, segment decomposition is derived data. The archive therefore keeps the rationale for dropping the 13 persisted grammar_* fields from StandardName nodes and replacing them with parser-version and diagnostics metadata.

  • Why drop the fields: duplicated truth, silent staleness when the parser changes, schema bloat, and sparse real usage.
  • What replaces them: on-demand parsing for search/audit paths, a grammar_parse_version stamp, and structured diagnostics instead of ad-hoc validation strings.
  • What downstream work depended on this: prompt rewrites, exemplar refresh, reviewer-rubric updates, and bootstrap validation against a regenerated corpus.

§3 — Why it is archived instead of treated as shipped

The source explicitly described itself as draft and placed hard gates in front of codex adoption: rc21 had to be released from ISN first, then codex could simplify schema and prompts against a stable grammar surface. That makes this file historical design context, not a claim that the full grammar-vNext migration landed exactly as written.

Keep this archive as the record of the intended sequence: preserve the corpus, formalize the grammar, release rc21, then simplify codex storage and prompt logic against the released parser.
