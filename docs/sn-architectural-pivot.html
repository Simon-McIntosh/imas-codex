SN Architectural Pivot

This HTML migration preserves the state of plan 29 as a partially landed architectural pivot. The implementation fraction remains 0.60: the generate/enrich split, lifecycle redesign, and most of the schema-contract work have landed conceptually, while the clean-slate rotation and grammar-graph tail are still open.

§1 — What this pivot locked in

  • Generate/enrich split. sn generate became the names-only stage and sn enrich was established as the standalone documentation pass.
  • Lifecycle separation. The plan replaced the overloaded drafted concept with the explicit named → enriched → reviewable path.
  • Grammar upgrade direction. The plan fixed the Transformation enum drift, introduced decomposition and over_ region handling, and deferred basis-frame / tensor / component-split follow-ons.
  • Qualifier-axis policy. The wave-absorbed-power family moved from name proliferation toward structural qualifier fields on StandardName.
  • Grammar-graph boundary. ISN remained the grammar source of truth and export surface, while imas-codex kept LinkML ownership for graph persistence.

§3 — Blockers and active risks

  • Embedding-null blocker. The nearby-standard-name enrichment surface collapses if name and description embeddings are not populated.
  • Rotation not yet proven. The plan still expects the destructive clear, seven-domain generate/enrich rotations, and the reviewer sign-off cycle to happen before the pivot is truly closed.
  • Grammar-graph coupling remains open. The sync path between ISN grammar exports and codex graph state is still the principal unresolved tail.
  • Inter-repo release latency. The ISN release cadence still gates the codex phases that depend on a pinned grammar rc.

§4 — Dependency metadata

Plan 29 is the architectural predecessor for the quality-bootstrap and prompt/fan-out plans. It directly blocks plan 31 and materially informs plans 32 and 39.

§5 — 2026-06-05 status update

Reality check from the live repo: phases A-C are already embodied in the current codebase, and enough of the former grammar-as-graph tail is present that the old “Phase E open” wording is now too coarse. This session also landed the legacy derived-parent lifecycle repair and removed the final invalid bare-base parent from the live graph, which brought the SN graph-quality suite back to green.

The plan nevertheless stays active. The remaining honest closure work is operational rather than architectural: run and record the bounded clean-slate rotation evidence, then either explicitly carve any surviving grammar-as-graph tail out of this plan or close the pivot with dated proof.

That means the bootstrap plan is no longer blocked on the derived-parent defect, but this pivot still is not honestly “done” until the rotation proof is written down.
