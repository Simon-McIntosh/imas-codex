§2 — Axis A: embedding-cache consistency (DD)

The DD embed worker (dd_workers.py) claims paths through claim_paths_for_embedding() (dd_graph_ops.py:332), which filters on status='refined'. Legacy classified nodes sit past that status, so once they lost (or never gained) a vector they are permanently invisible to the embed loop. Two mismatch directions exist:

  • ~2792 with embedding_text, no vector — of which ~2457 are legacy classified with enrichment_source=None (a state the current pipeline cannot produce). These need either a reset to refined so the embed worker reclaims them, or explicit exclusion if their node_category is genuinely non-embeddable (infra metadata such as ids_properties/*, code/*).
  • ~198 with a vector, no embedding_text — the embed step should persist the source text alongside the vector; backfill embedding_text for these on (re-)embed so the cache is self-consistent in both directions.

First separate the ~2457 into should-embed vs non-embeddable-by-category (query node_category + enrichment_source). Non-embeddable categories should be excluded from the completeness assertion at its source, not force-embedded; genuinely missing embeddings are reset to refined and drained through imas dd (the cheapest faithful path — resume, don't full-rebuild). Regenerate through the backing pipeline; never patch vectors or embedding_text in-place via Cypher.

§3 — Axis B: SN-side description & embedding coverage

Entirely on the standard-names side, not DD. 14 StandardName and 203 StandardNameSource nodes carry empty descriptions, and 15/4028 StandardName nodes are unembedded. The fix is to run these through the SN pipeline (sn run) rather than hand-writing text: regenerate the missing descriptions and let the embed step cover the 15 unembedded names. Before regenerating, triage whether any of the 203 empty-description sources are stale (superseded / orphaned) and should be pruned instead of regenerated — an empty description on a dead source is noise, not a gap.

§4 — Axis C: COCOS half-state on removed nodes (DD)

20 nodes fail test_no_cocos_half_state: they retained cocos_label_source but lost cocos_transformation_type when the build's stale-label clear pass (build_dd.py ~line 1015) ran. All 20 are lifecycle_status='removed' — historical fields that no longer exist in the current DD horizon. The dd-horizon-update plan already locked D-historical = current-only (historical/removed nodes are retained for provenance but are not held to current-node invariants). That decision points strongly at scoping the test to lifecycle_status <> 'removed' rather than grooming dead nodes — but grooming (clear cocos_label_source whenever the build clears cocos_transformation_type, so removed nodes never enter a half-state) is the alternative. See the decision below.
