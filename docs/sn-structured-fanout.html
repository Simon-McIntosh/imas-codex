Structured Fan-Out for SN Compose Pipeline

This HTML migration preserves plan 39 as a v3 draft. The implementation fraction remains 0.05: the proposer / executor / synthesizer design, helper prerequisites, telemetry contract, and rollout gate are all specified, but the code was explicitly still plan-only.

§1 — Fixed design constraints

  • Non-agentic only. The plan stays bounded, schema-validated, and fixed to exactly two LLM calls.
  • Depth fixed at one. The plan intentionally forbids depth-2 chaining and runtime function generation.
  • Closed catalog. Fan-out calls come from a discriminated-union schema over a small helper set.
  • True no-op failure semantics. Any planner, executor, budget, or disabled-path failure falls back to the baseline refine prompt with empty evidence.
  • Telemetry gate. Future fan-out expansion is explicitly blocked on Phase 1 within-cohort A/B evidence and the telemetry appendix.

§3 — Current blockers

  • Phase 0 helper work is mandatory. The catalog assumes public sync helpers in imas_codex.graph.dd_search with caller-supplied GraphClient.
  • Telemetry gate is binding. The Phase 2 / 3 rollout is intentionally blocked on the Phase 1 telemetry appendix and CI lint enforcement.
  • Budget accounting must stay attributable. The plan rewrites the fan-out budget invariant so that every extension is tied to explicit sub-events.
